# 국내 ETF 실제 거래가격 지원 설계

## 배경

국내주식 Property Inspector는 종목코드를 숫자 6자리로만 검증한다. 따라서 `0210A0`처럼 영문이 포함된 국내 ETF 종목코드를 저장할 수 없다. 또한 초기 가격 스냅샷은 모든 국내 상품에 일반 주식 현재가 REST API를 사용하고 있다.

KIS는 ETF/ETN 현재가용 REST API를 별도로 제공하지만, ETF의 실제 실시간 체결가격은 일반 국내 상장상품과 동일한 통합 실시간체결가 웹소켓으로 받을 수 있다. ETF 전용 실시간 NAV는 이번 기능이 표시하려는 실제 거래가격과 다르므로 사용하지 않는다.

## 목표

- 숫자 또는 영문 대문자로 구성된 6자리 국내 종목코드를 지원한다.
- 사용자가 국내 상품 유형을 `주식` 또는 `ETF/ETN`으로 선택할 수 있게 한다.
- ETF/ETN의 초기 스냅샷은 ETF/ETN 현재가 REST API에서 조회한다.
- 주식과 ETF/ETN 모두 기존 통합 실시간체결가 웹소켓에서 실제 거래가격을 갱신한다.
- 상품 유형 설정이 없는 기존 버튼은 일반 주식으로 동작해 하위 호환성을 유지한다.

## 비목표

- ETF의 iNAV, 괴리율, 추적오차 또는 구성 종목을 카드에 표시하지 않는다.
- 종목 마스터 파일을 내려받아 상품 유형을 자동 판별하지 않는다.
- 기존 카드 레이아웃이나 가격 표시 형식을 바꾸지 않는다.
- NXT/KRX 거래소를 사용자가 별도로 선택하게 하지 않는다.

## 검토한 접근

### 1. 영숫자 코드만 허용하고 기존 API 유지

변경 범위는 가장 작지만 ETF 전용 REST API를 사용하지 않아 상품별 API 경계가 불명확하다. ETF 응답 호환성에 의존하게 되므로 채택하지 않는다.

### 2. 상품 유형을 명시적으로 선택하고 REST API를 분기

Property Inspector에 `주식`과 `ETF/ETN` 선택을 추가한다. 초기 REST 조회만 상품 유형에 따라 분기하고 실시간 체결 스트림은 공유한다. 동작이 명확하고 불필요한 재시도 호출이 없어 이 방식을 채택한다.

### 3. 일반 주식 REST 실패 시 ETF REST로 자동 폴백

기존 UI를 유지할 수 있지만 잘못된 종목, 네트워크 오류, 인증 오류와 상품 유형 불일치를 구분하기 어렵다. 실패 시 API 호출도 늘어나므로 채택하지 않는다.

## 설정과 하위 호환성

`DomesticStockSettings`에 선택적 `instrumentType`을 추가한다.

- `stock`: 일반 주식
- `etf`: ETF/ETN
- 누락: 기존 설정과의 호환을 위해 `stock`으로 처리

Property Inspector의 상품 유형 기본값도 `stock`으로 한다. 종목코드는 `/^[0-9A-Z]{6}$/i`로 검증하고 저장 시 앞뒤 공백을 제거한 뒤 대문자로 변환한다. 안내 문구는 숫자 전용 표현을 제거하고 `005930`, `0210A0` 예시를 함께 보여준다.

런타임에서도 설정을 신뢰하지 않고 종목코드를 `trim().toUpperCase()`로 정규화한다. 이렇게 하면 기존에 소문자로 저장된 설정이나 외부에서 주입된 설정도 동일하게 처리된다.

## 데이터 흐름

1. 국내 액션이 설정에서 상품 유형과 종목코드를 읽는다.
2. 종목코드를 대문자로 정규화한다.
3. 초기 스냅샷과 수동 새로고침 및 폴링은 상품 유형을 REST 계층에 전달한다.
4. 일반 주식은 기존 `/uapi/domestic-stock/v1/quotations/inquire-price`, `FHKST01010100`을 사용한다.
5. ETF/ETN은 `/uapi/etfetn/v1/quotations/inquire-price`, `FHPST02400000`을 사용한다.
6. 두 REST 응답에서 카드에 필요한 공통 필드인 현재가, 전일 대비, 등락 부호, 등락률을 같은 `StockData`로 매핑한다.
7. 두 상품 유형 모두 기존 `H0UNCNT0`에 정규화된 종목코드로 구독한다.
8. 웹소켓 수신 데이터의 종목코드도 대문자로 정규화한 뒤 구독과 매칭한다.
9. 기존 국내 파서와 카드 렌더러로 실제 체결가격을 표시한다.

## 컴포넌트 변경

### Property Inspector

`ui/domestic-stock-pi.html`에 상품 유형 선택을 추가하고 종목코드 검증과 직렬화를 영숫자 코드에 맞게 변경한다.

### 타입과 상수

`DomesticStockSettings`에 `instrumentType`을 추가하고 ETF/ETN REST TR ID 상수를 정의한다. 상품 유형은 문자열 리터럴 유니온으로 제한한다.

### REST 현재가 조회

국내 현재가 조회 함수가 상품 유형을 입력받아 URL과 TR ID를 선택한다. 상품 유형 인자는 기본값을 `stock`으로 둬 기존 호출과 테스트의 호환성을 유지한다. 공통 응답 파싱과 오류 분류는 중복하지 않는다.

### 국내 액션

액션 생명주기, 설정 변경, 수동 새로고침, 초기 재시도, 폴링의 모든 REST 호출에 상품 유형을 전달한다. 웹소켓 TR ID와 파서는 변경하지 않는다. 설정 변경 시 상품 유형만 바뀌어도 기존 구독과 런타임 상태를 재설정하고 새 REST 경로로 조회한다.

### 웹소켓 매칭

국내 수신 종목코드를 대문자로 정규화해 `0210A0` 같은 영숫자 코드가 구독 키와 정확히 매칭되게 한다.

## 오류 처리

- 인증, 네트워크, 유효하지 않은 종목 오류는 기존 `ErrorType` 분류와 카드를 그대로 사용한다.
- 상품 유형과 종목이 맞지 않아 REST 응답에 현재가가 없으면 `INVALID_STOCK`으로 처리한다.
- REST 스냅샷이 실패해도 기존 정책대로 웹소켓 구독은 계속 시도한다.
- ETF 전용 REST API에 별도 폴백을 추가하지 않는다.

## 테스트 전략

테스트는 TDD 순서로 추가한다.

1. 일반 주식 상품 유형이 기존 REST URL과 TR ID를 사용하는 테스트.
2. ETF 상품 유형이 ETF/ETN REST URL과 `FHPST02400000`을 사용하고 `0210A0`을 그대로 전달하는 테스트.
3. 소문자 `0210a0`이 런타임에서 `0210A0`으로 정규화되는 테스트.
4. 국내 액션이 ETF 설정을 초기 조회, 수동 새로고침, 재시도, 폴링 경로에 전달하는 테스트.
5. 통합 웹소켓이 `0210A0` 수신 데이터를 같은 코드의 구독에 전달하는 테스트.
6. 기존 전체 테스트와 프로덕션 빌드를 실행해 회귀를 확인한다.

Property Inspector는 현재 별도 DOM 테스트 환경이 없으므로 정규식과 직렬화 변경은 코드 검토로 확인하고, 런타임 정규화는 TypeScript 단위 테스트로 보호한다.

## 완료 조건

- Property Inspector에서 `0210A0`을 유효한 종목코드로 저장할 수 있다.
- ETF/ETN 선택 시 초기·수동·폴링 REST 조회가 ETF/ETN 현재가 API를 사용한다.
- `0210A0`의 실시간 실제 체결가격이 기존 통합 웹소켓을 통해 카드에 표시된다.
- 기존 숫자 6자리 주식 설정과 상품 유형이 없는 기존 버튼이 이전과 동일하게 동작한다.
- 전체 테스트와 빌드가 성공한다.
