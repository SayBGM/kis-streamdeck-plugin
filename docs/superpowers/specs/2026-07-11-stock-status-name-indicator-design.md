# 종목명 기반 시세 상태 표시 설계

## 배경

현재 주식 카드의 연결 상태는 카드 하단 4px 상태 바와 하단 상태 문구로 표시한다. `LIVE`는 초록, `BACKUP`은 노랑, `BROKEN`은 빨강이다. 장 마감이나 휴장 중 정상적으로 REST 스냅샷을 표시하는 카드도 노란색으로 보일 수 있어 사용자가 이를 경고 또는 고장으로 오해하기 쉽다.

종목명은 현재 기본 흰색이며, `stale`일 때만 노란색으로 바뀐다. 이 영역을 주 연결 상태 표시로 사용하면 Stream Deck의 작은 144×144 화면에서 시선을 카드 하단까지 옮기지 않고도 상태를 확인할 수 있다.

## 목표

- 연결 상태를 종목명 색상과 종목명 앞의 작은 점으로 표시한다.
- 정상적인 REST 백업과 장 마감 스냅샷을 경고색이 아닌 정보색으로 표현한다.
- 모든 카드에서 기존 하단 상태 바를 제거한다.
- `stale`이 종목명 상태색을 덮어쓰지 않게 한다.
- 실패와 지연 상태는 사용자가 이해할 수 있는 짧은 하단 문구로 보완한다.
- 시세 수집, 연결 상태 머신, REST/WebSocket 전환 정책은 변경하지 않는다.

## 비목표

- `StockActionView`의 상태 값이나 전환 규칙을 변경하지 않는다.
- 카드의 가격, 등락률, 장 세션 판정 또는 세션 배지 구조를 변경하지 않는다.
- 사용자가 상태 색상을 설정하는 옵션을 추가하지 않는다.
- 애니메이션이나 신규 아이콘 라이브러리를 추가하지 않는다.
- Property Inspector 또는 진단 화면을 변경하지 않는다.

## 검토한 접근

### 1. 종목명 색상만 사용

가장 간결하지만 작은 화면이나 빠른 시선 이동 상황에서 색상 변화가 눈에 덜 띌 수 있다. 상태 위치를 알려 주는 형태 단서도 없으므로 채택하지 않는다.

### 2. 종목명 색상과 작은 점 사용

종목명 바로 앞에 같은 상태색의 작은 원을 배치한다. 하단 바보다 시선 이동이 적고 긴 배지보다 공간을 적게 사용한다. 사용자가 선택한 방식이므로 채택한다.

### 3. 종목명 색상과 상태 배지 사용

`LIVE`, `REST`, `!` 배지는 의미가 가장 직접적이지만 세션 배지와 경쟁하고 긴 종목명의 사용 가능 폭을 줄인다. 144px 카드에는 시각적 밀도가 높아 채택하지 않는다.

## 상태별 시각 규칙

| 상태 | 종목명과 점 | 하단 상태 문구 | 의미 |
|---|---|---|---|
| `LIVE` | 초록 `#00c853` | `실시간` | WebSocket 실시간 시세가 정상 수신됨 |
| `BACKUP` | 파랑 `#7dd3fc` | `백업` | REST가 정상적으로 시세를 제공함 |
| `BROKEN` | 빨강 `#ff1744` | `연결 확인 필요` | 실시간과 REST 모두 정상 시세를 제공하지 못함 |
| `waiting` | 회색 `#7f8aa8` | 기존 대기 문구 | 초기 연결 또는 데이터 수신을 기다림 |

장 마감이나 휴장 중 REST로 얻은 정상 스냅샷은 상태 머신이 제공하는 연결 상태를 그대로 사용한다. `BACKUP`이면 파랑으로 표시하며, 노란 경고색을 사용하지 않는다.

`stale`은 연결 상태와 독립된 데이터 신선도 상태다. 종목명과 점의 색은 위 표의 연결 상태색을 유지하고 하단 문구만 노랑 `#ffd54f`의 `시세 지연` 또는 현재 상태와 결합한 `백업 · 지연`으로 표시한다. `refreshing`은 기존 로딩 표시와 `새로고침 중` 문구를 유지하며 종목명 상태색을 덮어쓰지 않는다.

## 카드 레이아웃

종목명이 있는 카드는 제목 행을 다음과 같이 구성한다.

- 제목 행 왼쪽에 반지름 약 3px의 원을 배치하고 `data-role="connection-dot"`을 부여한다.
- 종목명 텍스트는 점 오른쪽으로 이동하고 점과 같은 연결 상태색을 사용한다.
- 종목명 자르기, 글꼴 크기 계산, 세션 배지 위치는 기존 규칙을 유지한다.
- 하단 상태 문구는 기존 위치를 사용하되 상태 바가 사라진 공간과 겹치지 않게 한다.
- `data-role="state-bar"` 및 `CONNECTION_LINE_*` 기반 하단 사각형은 제거한다.

일반 시세 카드, 데이터 대기 카드, 오류 카드, 회복 카드에 같은 제목 행 규칙을 적용한다. 회복 카드는 `LIVE`의 초록 종목명과 점을 사용한다. 종목명이 없는 설정 필요·표시 오류 전용 카드는 점을 억지로 추가하지 않고 기존 중앙 아이콘과 문구를 유지하되 하단 상태 바만 제거한다.

레거시 공개 렌더 함수도 같은 시각 규칙을 따른다.

- `renderStockCard`: `connectionState`로 종목명과 점의 색을 결정한다.
- `renderWaitingCard`: 회색 종목명과 점을 사용한다.
- `renderConnectedCard`: 장중 연결 대기는 초록, 장 마감 대기는 파랑으로 표현한다.
- `renderRecoveryCard`: 초록 종목명과 점을 사용하고 하단 바를 제거한다.
- `renderErrorCard`, `renderSetupCard`: 종목명이 없으므로 기존 전용 화면을 유지한다.

## 렌더러 구조

`src/renderer/stock-card.ts` 안에 연결 상태를 제목 표시 정보로 변환하는 작은 헬퍼를 둔다. 헬퍼는 상태별 색상과 점 SVG를 한곳에서 결정하고 `StockActionView` 경로와 레거시 공개 함수가 공유한다.

렌더러는 전달받은 상태만 사용하며 현재 시간이나 외부 연결 객체를 읽지 않는다. 따라서 동일한 화면 모델은 이전과 마찬가지로 결정적인 SVG를 생성한다. 액션 컨트롤러, 렌더 스케줄러, 캐시 키 구조에는 새 상태 필드를 추가하지 않는다. 상태가 이미 렌더 의미 키에 포함되어 있으므로 색상과 점 변경도 기존 캐시 무효화 규칙으로 처리된다.

## 오류 및 조합 상태

- 오류 코드가 있는 종목 카드의 종목명과 점은 빨강으로 표시한다.
- 가격이 있는 `BROKEN` 화면은 가격과 등락 정보를 유지하고 하단에 `연결 확인 필요`를 표시한다.
- 가격이 없는 `BROKEN` 화면은 기존 오류 또는 연결 대기 내용을 유지한다.
- `BACKUP + stale`은 파란 종목명과 점, 노란 `백업 · 지연` 문구로 표현한다.
- `LIVE + stale`은 초록 종목명과 점, 노란 `시세 지연` 문구로 표현한다.
- `refreshing`은 연결 상태색의 종목명과 점, 기존 로딩 표시와 `새로고침 중` 문구를 사용한다.
- 입력 검증 실패로 생성되는 `표시 오류` 카드는 종목명이 없으므로 빨간 중앙 아이콘을 유지하고 하단 바만 제거한다.

## 테스트 전략

렌더러 변경은 TDD 순서로 구현한다.

1. `LIVE`, `BACKUP`, `BROKEN`, `waiting`별로 `data-role="connection-dot"`과 기대 색상의 종목명/점이 생성되는 실패 테스트를 추가한다.
2. 모든 `StockActionView` 분기에서 `data-role="state-bar"`가 생성되지 않는 실패 테스트를 추가한다.
3. `BACKUP + stale`과 `LIVE + stale`에서 종목명 상태색은 유지되고 지연 문구만 노란색으로 표시되는 테스트를 추가한다.
4. `BROKEN`의 하단 문구가 `연결 확인 필요`로 표시되는 테스트를 추가한다.
5. `renderStockCard`, `renderWaitingCard`, `renderConnectedCard`, `renderRecoveryCard`의 제목 표시와 하단 바 제거를 검증한다.
6. 기존 SVG 파싱, XML 이스케이프, 결정성, 캐시 테스트를 유지한다.
7. 렌더러 관련 테스트, 전체 Vitest 테스트, 프로덕션 빌드를 실행한다.

테스트는 문자열 전체 스냅샷보다 `data-role`과 상태별 색상·문구를 검증해 레이아웃의 사소한 변경에 과도하게 결합하지 않게 한다.

## 완료 조건

- 일반 시세 카드에서 종목명과 작은 점만으로 `LIVE`, `BACKUP`, `BROKEN`, `waiting`을 구분할 수 있다.
- `BACKUP`과 장 마감 REST 스냅샷은 노랑이 아닌 파랑으로 표시된다.
- `stale`이 종목명 연결 상태색을 변경하지 않는다.
- 연결 실패는 빨간 종목명과 점, `연결 확인 필요` 문구로 표시된다.
- 어떤 렌더링 경로에서도 기존 하단 상태 바가 생성되지 않는다.
- 설정·인증·종목 오류 전용 화면의 의미는 유지된다.
- 전체 테스트와 프로덕션 빌드가 성공한다.
